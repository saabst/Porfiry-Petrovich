# 🕵️ Порфирий Петрович — Литературный ИИ-следователь

> Алгоритмический следователь, который ведёт диалог с пользователем и параллельно пишет роман в стиле Виктора Пелевина или Фёдора Достоевского. Работает на базе **Xiaozhi + ESP32-S3**, сохраняет сырые заметки в Google Docs, а скомпилированные главы — в красивые PDF книги с иллюстрациями.

![Архитектура проекта](image.png)

## ✨ Ключевые возможности

### 📝 **Интеллектуальная компиляция**
- **Накопление заметок**: собирает сырые записи до порога (по умолчанию 12)
- **Автоматическая компиляция**: генерирует связные главы через DeepSeek API
- **Контекстное понимание**: учитывает предыдущие главы и темы
- **Асинхронная обработка**: не блокирует приём новых заметок

### 🎭 **Мульти-стилевая система**
- **Стиль Пелевина**: метафизика, цифровой детектив, постмодернизм
- **Стиль Достоевского**: психологизм, экзистенциальные вопросы, глубина характеров
- **Конфигурируемые стили**: YAML-файлы в папке `styles/`
- **Динамическое переключение**: можно менять стиль через переменные окружения

### 🎨 **Визуализация и экспорт**
- **Автоматические иллюстрации**: генерация картинок к каждой главе через Pollinations.ai
- **Граф знаний**: извлечение фактов (субъект-предикат-объект) и визуализация
- **PDF-экспорт**: создание красивых PDF книг через WeasyPrint
- **Markdown-экспорт**: сохранение в удобном для чтения формате

### 🔗 **Интеграции**
- **Google Docs**: автоматическая запись глав и заметок
- **Xiaozhi MCP**: голосовой интерфейс через ESP32-S3
- **DeepSeek API**: мощная генерация текста
- **Pollinations.ai**: бесплатная генерация изображений

## 🚀 Быстрый старт (5 минут)

```bash
# 1. Клонирование и настройка
git clone https://github.com/saabst/Porfiry-Petrovich.git
cd Porfiry-Petrovich
python3 -m venv venv
source venv/bin/activate  # Linux/Mac
# venv\Scripts\activate   # Windows

# 2. Установка зависимостей
pip install -r requirements.txt

# 3. Базовая конфигурация
cat > .env << EOF
XIAOZHI_MCP_TOKEN=ваш_токен_xiaozhi
DEEPSEEK_API_KEY=ваш_ключ_deepseek
COMPILE_THRESHOLD=5  # Для быстрого тестирования
USE_IMAGES=true
USE_PDF=true
LOG_LEVEL=INFO
EOF

# 4. Запуск
python3 main.py
```

**Полная инструкция**: [ИНСТРУКЦИЯ_УСТАНОВКИ.md](ИНСТРУКЦИЯ_УСТАНОВКИ.md)

## 🎮 Использование

### Голосовой интерфейс (через Xiaozhi)
1. Подключите ESP32-S3 с прошивкой Xiaozhi
2. Говорите с устройством - заметки автоматически записываются
3. После накопления заметок запускается компиляция главы

### Пример рабочего процесса
```
Пользователь: "Сегодня утром я видел странную тень в окне"
Порфирий: ✅ Заметка L001-001 сохранена

Пользователь: "Она двигалась неестественно, как цифровой артефакт"
Порфирий: ✅ Заметка L001-002 сохранена

... после 12 заметок ...
Порфирий: 🔔 Порог достигнут, запуск компиляции...
Порфирий: 📘 Глава 1 успешно скомпилирована и записана в роман
```

**Больше примеров**: [ПРИМЕРЫ_ИСПОЛЬЗОВАНИЯ.md](ПРИМЕРЫ_ИСПОЛЬЗОВАНИЯ.md)

## 🔧 Расширенные настройки

### Конфигурация (.env)
```env

XIAOZHI_MCP_TOKEN= мсп ксиаоши
DEEPSEEK_API_KEY= ключ дипсик
DEEPSEEK_MODEL=deepseek-chat
DIARY_DOC_ID= айди дневника (гугл док)
NOVEL_DOC_ID= айди романа (гугл док)
GOOGLE_CREDENTIALS_FILE=credentials.json - авторизация гугл сервисная
COMPILE_THRESHOLD=6 #Сколько сообщений диалога для компиляции главы

# Дополнительные переменные (режим работы)
AUTHOR_STYLE=pelevin           # dostoevsky, pelevin, или путь к своему yaml
USE_GRAPH=false                 # включить граф знаний
USE_IMAGES=true                # генерировать иллюстрации
OUTPUT_DIR=output              # папка для сохранения markdown, images, graph
TIMEZONE=Europe/Moscow

USE_PDF=true
PDF_AFTER_CHAPTER=2
PDF_OUTPUT=book.pdf

# Настройки графа
GRAPH_STATE_FILE=knowledge_graph.json

# Настройки генерации иллюстраций
IMAGE_PROVIDER=pollinations    # pollinations или none
IMAGE_WIDTH=1024
IMAGE_HEIGHT=1024

# Логирование
LOG_LEVEL=INFO
```

### Настройка стилей авторов
Стили настраиваются через YAML-файлы в папке `styles/`:

**Пример `pelevin.yaml`:**
```yaml
name: "pelevin"
description: "Стиль Виктора Пелевина — метафизика, цифровой детектив"
system_prompt: |
  Ты — Порфирий Петрович, следователь в цифровом ГУЛАГе...
user_prompt_template: |
  Собери главу из заметок:
  {entries}
temperature: 0.7
max_tokens: 2000
```

## 📈 Что вы получите?

### После компиляции каждой главы:
1. **Текст главы** в стиле выбранного автора
2. **Иллюстрация** на основе содержания главы
3. **Обновление графа знаний** с новыми фактами
4. **Запись в Google Docs** (если настроено)
5. **Обновление Markdown файла**
6. **PDF книга** (после указанного количества глав)

### Пример сгенерированного контента:
```
ГЛАВА 1
МЕТАФИЗИКА КОДА

Тень в окне была не просто отсутствием света, а активным отрицанием реальности. 
Цифровой артефакт, просочившийся сквозь щели рендеринга...

ПРИМЕЧАНИЯ СЛЕДОВАТЕЛЯ
[L001-001]: Тень как метафора цифрового бессознательного
[L001-002]: Артефакты рендеринга как сбои в матрице восприятия
```

## 🐛 Отладка и решение проблем

### Частые ошибки и решения

**Ошибка подключения к Xiaozhi:**
```bash
# Проверьте токен
echo $XIAOZHI_MCP_TOKEN

# Проверьте подключение
curl -I https://api.xiaozhi.me
```

**Ошибка Google Docs API:**
```bash
# Проверьте credentials.json
cat credentials.json | head -5

# Проверьте ID документов
echo "DIARY: $DIARY_DOC_ID"
echo "NOVEL: $NOVEL_DOC_ID"
```

**Быстрые команды для проверки:**
```bash
# Проверка состояния системы
cat story_state.json

# Просмотр кэша заметок
cat diary_cache.json

# Проверка сгенерированных файлов
ls -la output/
```

**Полное руководство по отладке**: [БЫСТРЫЙ_СТАРТ.md](БЫСТРЫЙ_СТАРТ.md)
